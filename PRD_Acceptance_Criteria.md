# QGIS Agent 插件 — 产品需求文档 (PRD) & 验收标准

**产品名称：** QGIS Agent
**版本：** 1.0.0
**目标平台：** QGIS 4.0.0-1 (Windows / macOS / Linux)
**文档版本：** 1.0
**日期：** 2026-05-08

---

## 1. 产品概述

### 1.1 产品定位

QGIS Agent 是一个以侧边栏形式嵌入 QGIS 4.0 的 AI 智能助手插件。用户通过自然语言描述 GIS 操作需求，AI 自动拆解为子任务并调用对应的 GIS 工具完成操作，消除繁琐的手动步骤。

### 1.2 目标用户

| 用户类型 | 痛点 | 期望 |
|----------|------|------|
| GIS 分析师 | 批量操作重复繁琐（逐图层转坐标系、逐文件裁剪） | 一句话完成批量操作 |
| GIS 初学者 | 不熟悉 Processing 工具箱参数 | 自然语言描述即可，无需记忆算法名和参数 |
| 项目经理 | 需要快速查看统计结果和数据概况 | 快速获取图层统计信息 |

### 1.3 核心价值

- **效率提升**：将多步手动操作缩减为一句自然语言指令
- **降低门槛**：无需记忆 QGIS Processing 算法名和参数
- **智能拆解**：复杂需求自动分解为可执行的子任务序列

## 4. 用户故事 & 场景验收

### US-01: 批量坐标转换

> **作为** GIS 分析师
> **我希望** 一句话将 10 个图层从 WGS84 转到 CGCS2000
> **这样我不用** 逐个打开图层属性、设置目标 CRS、运行重投影

**验收场景：**
1. 打开包含 roads、rivers、buildings、parks、landuse 等 10 个 WGS84 图层的 QGIS 项目
2. 在 QGIS Agent 侧边栏输入："把所有图层转成 CGCS2000"
3. **预期：** AI 回复 "我将把 10 个图层批量转换到 CGCS2000 (EPSG:4490)"，显示任务进度
4. **预期：** 10 个工具卡片依次显示执行状态，全部绿色完成
5. **预期：** 图层面板中 10 个图层的 CRS 全部变为 EPSG:4490
6. **预期：** 要素数量转换前后一致

### US-02: 复合任务拆解

> **作为** GIS 初学者
> **我希望** 用一句话完成"画缓冲区 + 裁剪"
> **这样我不用** 分别学两个工具的用法

**验收场景：**
1. 打开包含 roads 和 admin_boundary 图层的项目
2. 输入："在 roads 周围创建 500 米缓冲区，然后用 admin_boundary 裁剪"
3. **预期：** AI 拆解为 2 个步骤，显示任务进度列表
4. **预期：** 步骤 1 完成后生成 buffer 图层，步骤 2 裁剪后生成最终图层
5. **预期：** 最终结果只包含行政边界内的道路缓冲区

### US-03: 模糊需求处理

> **作为** 用户
> **我希望** 输入模糊指令时 AI 会追问而不是瞎猜
> **这样我不用** 担心 AI 执行错误操作

**验收场景：**
1. 输入："帮我处理一下数据"
2. **预期：** AI 回复询问具体需求，例如"您希望对哪些图层做什么操作？目前项目中有以下图层：..."
3. **预期：** 不会触发任何工具执行

### US-04: 切换大模型

> **作为** 用户
> **我希望** 能在 DeepSeek 和通义千问之间切换
> **这样我可以** 对比不同模型的效果

**验收场景：**
1. 打开设置对话框，选择 DeepSeek Provider，输入 API Key，保存
2. 发送消息，验证正常回复
3. 打开设置对话框，切换到通义千问 Provider，输入 API Key，保存
4. 发送相同消息，验证正常回复
5. **预期：** 两次回复均正常，工具调用均正确

### US-05: 错误恢复

> **作为** 用户
> **我希望** 工具执行失败时能知道原因并重试
> **这样我不用** 从头开始

**验收场景：**
1. 输入对一个不存在的图层执行操作
2. **预期：** AI 报错 "图层 'xxx' 在当前项目中不存在"，列出可用图层
3. 输入正确的图层名重试
4. **预期：** 正常执行成功

### US-06: 双源输入

> **作为** 用户
> **我希望** 能通过插件侧边栏直接添加文件，而不必先在 QGIS 中加载
> **这样我不用** 在 QGIS 和文件管理器之间来回切换

**验收场景：**
1. 打开一个空的 QGIS 项目（无图层）
2. 点击侧边栏输入框左侧的📎按钮，选择 roads.shp
3. 输入："把 roads 转成 3857"
4. **预期：** 文件自动加载为临时图层"[plugin] roads"，AI 执行坐标转换
5. **预期：** 转换结果添加到项目中

**混合数据源场景：**
1. QGIS 项目中已有 rivers 图层
2. 通过📎按钮添加 roads.shp
3. 输入："把所有图层转成 3857"
4. **预期：** AI 追问"检测到两种数据来源：项目中有 rivers，您导入了 roads。请问使用哪种？"
5. 回答"全部"
6. **预期：** 两个图层都被转换

## 6. 术语表

| 术语 | 定义 |
|------|------|
| CRS | 坐标参考系统 (Coordinate Reference System)，如 EPSG:4326 |

| OpenAI 兼容协议 | 遵循 OpenAI Chat Completions API 格式的接口规范 |
| SSE | Server-Sent Events，流式 HTTP 响应协议 |
| Processing.run() | QGIS Processing 框架执行算法的 Python API |
| QDockWidget | Qt 可停靠窗口组件，用于实现侧边栏 |
| Tool Card | UI 中展示工具调用参数和状态的可折叠卡片组件 |
